Indic Language and Script "On Device" Tests

This page can be accessed at http://htmlpreview.github.io/?https://github.com/googlei18n/noto-source/blob/main/test/BIS-Indic-ondevice-tests.html

Open this page and and all the language/script relevant links on a device that you'd like to test your fonts on.

Font rendering issues could have one of these two symptoms:

  • If you see an empty rectangle similar to □ ("tofu") instead of a proper character then it means that the font on your device doesn't support a character required for this language. This situation should be noted (please make sure that you save the text that caused it somewhere) and brought up to the attention of the phone/device manufacturer or a font provider.
  • If you see a character rendered by a browser not recognizable by a native reader on a the web page then this situation also should be noted (please make sure that you save the text that caused it somewhere) and brought up to the attention of the phone/device manufacturer or a font provider.
Who can test what:
  • If you are not a native speaker/reader then you can inspect all the language/script relevant pages for a presence of an empty rectangle similar to □ ("tofu") instead of a proper character (the first item in the above list). This most likely indicates a problem with the font on your device.
  • If you are a native speaker/reader then you can assess how correct and readable the characters or sample texts on the web pages available here are.

The fonts used by your browser to display these pages are the default fonts on the phone or the device you are using. If this is a standard Android device which wasn't modified by a phone manufacturer then this non-Latin font belongs to the Noto family (see get/noto for more information).


The table below has the following columns and related test possibilities:
  • IS 16350 Table: Passing correctness and readability for this table is required by Bureau of Indian Standards (BIS). This is the table of characters required by IS 16350 Standard for a given language in India. When open on a phone or a device, the table will reflect the default fonts loaded on this device. Black boxes in the table indicate a character that wasn't required or doesn't exist in the IS 16350 standard. Please check if the web page is readable and please make sure that what you see is correctly rendered. Please make a note of any text that's not readable or incorrect.
  • IS 16333 (Part 3) Table: Bengali, Devanagari, Gurmukhi and Oriya scripts have additional requirements for conjuncts. Passing correctness and readability for this table is required by BIS. Please check if the web page is readable and please make sure that what you see is correctly rendered. In any given row of the table, all the non-English script characters in that row ought to be exactly the same. Please make a note of any text that's not readable or incorrect.
  • Language: identifies the language. If there's a wikipedia entry in a variant of this language then you'll see a link there. Please check if the web page is readable and please make sure that what you see is correctly rendered. Please make a note of any text that's not readable or incorrect.
  • ICA Requirements: is a set of characters identified by the Indian Cellular Association (ICA) as required to support a given language by a given script. Any "tofu" there needs to be reported. Same if any of the characters are incorrect or unreadable.
  • GSUB Tests: is a limited set of tests using character combinations that compose (SUBstitue to) a new character. Any "tofu" there needs to be reported. Same if any of the characters are incorrect or unreadable.
  • UDHR or Other Sample: UDHR stands for Universal Declaration of Human Rights. It is published by the Office of the United Nations High Commissioner for Human Rights (our copy comes from the UNICODE organization). UDHR is easily accessible sample text available for a language and script(font) combination. Please check the readability.
